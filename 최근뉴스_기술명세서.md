# 알람 웹앱 — 최신 뉴스 검색 및 음성 읽기 기술 명세서

## 개요

기존 알람 웹앱에 **최신 뉴스 5건 검색** 및 **음성 읽기(TTS)** 기능을 추가하기 위한 기술 명세서입니다.
핵심 스택으로 **React**(프론트엔드) + **Supabase**(백엔드/DB/인증)를 사용합니다.

---

## 1. 기술 스택 개요

| 계층 | 기술 | 역할 |
|---|---|---|
| 프론트엔드 | React (Vite + TypeScript) | UI 컴포넌트, 상태 관리, 라우팅 |
| 백엔드/DB | Supabase | PostgreSQL DB, Auth, Edge Functions, Realtime |
| 뉴스 검색 | Claude API + Web Search | Supabase Edge Function에서 호출 |
| 뉴스 대안 | NewsAPI.org | 간단한 REST API, 무료 플랜 |
| 음성 읽기 | Web Speech API | 브라우저 내장 TTS, 한국어 지원 |
| 음성 대안 | Google Cloud TTS / CLOVA Voice | 더 자연스러운 음질 |
| 스타일링 | Tailwind CSS | 유틸리티 기반 빠른 UI 구축 |

## 2. Supabase 활용 상세

### 2.1 데이터베이스 (PostgreSQL)

**alarms 테이블** — 알람 정보를 저장합니다. 사용자 ID, 시간, 라벨, 반복 요일(일~토), 활성 여부, 생성일시 컬럼으로 구성합니다.

**news_cache 테이블** — API 호출을 최소화하기 위해 뉴스를 캐싱합니다. 키워드, 제목, 2줄 요약, 출처 URL, 발행일, 조회일시 컬럼으로 구성하고, 1시간 이내 캐시만 유효하도록 인덱스를 설정합니다.

### 2.2 인증 (Supabase Auth)

Supabase Auth를 사용하여 사용자별 알람을 관리합니다. 이메일/비밀번호 인증은 물론 소셜 로그인(Google, Kakao 등)도 간편하게 추가할 수 있습니다.

### 2.3 Row Level Security (RLS)

사용자가 자신의 알람만 접근하도록 RLS 보안 정책을 설정합니다. 모든 CRUD 작업에서 auth.uid()와 user_id를 비교하여 본인 데이터만 조회·수정 가능하도록 제한합니다.

### 2.4 Edge Functions — 뉴스 검색 API

Supabase Edge Functions(Deno 기반)에서 Claude API를 호출합니다. API 키를 서버 측 환경 변수로 관리하여 클라이언트에 노출하지 않습니다. Claude API에 Web Search 도구를 활성화하고, 한국 최신 뉴스 5건을 JSON 형태(제목, 2줄 요약, 출처)로 요청합니다.

### 2.5 Realtime (선택)

Supabase Realtime을 활용하면 여러 기기에서 알람 상태를 실시간 동기화할 수 있습니다. alarms 테이블의 변경 이벤트를 구독하여 다른 기기에서도 즉시 반영됩니다.

## 3. React 프론트엔드 구조

### 3.1 프로젝트 구조

| 폴더/파일 | 역할 |
|---|---|
| components/AlarmClock | 시계 표시 + 알람 트리거 |
| components/AlarmForm | 알람 추가/편집 폼 |
| components/AlarmList | 알람 목록 |
| components/NewsCard | 개별 뉴스 카드 |
| components/NewsFeed | 뉴스 5건 피드 |
| components/TtsPlayer | 음성 읽기 컨트롤러 |
| hooks/useAlarms | 알람 CRUD (Supabase 연동) |
| hooks/useNews | 뉴스 검색 (Edge Function 호출) |
| hooks/useTts | Web Speech API 래핑 |
| lib/supabase | Supabase 클라이언트 초기화 |
| pages/Home | 메인 화면 (시계 + 알람) |
| pages/Login | 로그인/회원가입 |

### 3.2 핵심 커스텀 훅

**useNews** — Supabase Edge Function을 호출하여 뉴스를 가져오고, loading/news 상태를 관리합니다. 키워드를 인자로 받아 기본값은 "주요 뉴스"입니다.

**useTts** — Web Speech API를 래핑하여 개별 뉴스 읽기(speak), 전체 뉴스 순서대로 읽기(speakAll), 읽기 중지(stop) 기능을 제공합니다. 한국어(ko-KR) 음성을 사용하며, 현재 읽기 상태(isSpeaking)를 추적합니다.

---

## 4. 구현 흐름

알람 시간 도달 → Supabase Edge Function 호출 → Claude API + Web Search로 최신 뉴스 5건 검색 → news_cache 테이블에 캐싱 → React 프론트엔드에 뉴스 카드 5장 렌더링 → 사용자가 개별 뉴스 클릭 시 해당 뉴스만 TTS 읽기, 전체 읽기 버튼 클릭 시 5건 순서대로 TTS 읽기

---

## 5. 환경 변수 설정

**React (.env.local)** — VITE_SUPABASE_URL(Supabase 프로젝트 URL)과 VITE_SUPABASE_ANON_KEY(공개 키)를 설정합니다.

**Supabase Edge Function Secrets** — ANTHROPIC_API_KEY를 Supabase 대시보드에서 시크릿으로 등록합니다.

---

## 7. 배포

프론트엔드는 **Vercel** 또는 **Netlify**로 배포하고, 백엔드는 Supabase가 호스팅하므로 별도 서버가 필요 없습니다. Edge Function도 Supabase 내에서 자동 배포됩니다.
